Validating multiple tasks at once or “Multivalidation”

When you are reading this you should be already experienced with validating and using JOSM so I will not include basics here.

Usually I am validating projects in HOT Tasking Manager that are focused on mapping buildings. I am going to show you an example of my workflow on one of the Mozambique projects because there are two things that are not in every project and it’s very easy to validate in this area so you can try it on your own. Since writing this diary the campaign ended but you can find similar projects using this filter on the HOT Tasking Manager.

I highly recommend using the shortcuts in JOSM, which you can customize, or you can put tools that you are often using to your toolbar. I will write shortcuts for windows in {}.

For Multivalidation in Mozambique I am using these tools:

EXPERT MODE

Edit → Preferences {F12} → checkbox in the bottom left corner. Thanks to this you will be able to see some stuff that has been hidden before like the number of each layer, more options for Search or some tools like extrude tool for example. If you are using shortcuts (like {X} for extrude tool) those will work even if you won’t turn on the Expert mode.

PLUGINS

List of plugins that I am using and their most useful feature for me:

  • Buildings_tools - fundamental plugin for mapping buildings
  • Todo - great tool for adding buildings, task areas, anything into list and then easily check all of them one by one
  • Utilsplugin2 - mainly for Replace geometry {Ctrl+Shift+G}
  • Mapathoner - mainly for Selecting non orthogonal buildings
  • Markseen - mark seen areas of the map
VALIDATOR RESULTS PANEL

I am using additional custom rules that are flagging some uncommon mistakes made by other mappers. The name is "Validating tasks in HOT TM projects". Here is their full description with guide how to add them into JOSM.

Taskgrid

Importing borders of every task (taskgrid) in the project:

File › Open Location {Ctrl+L} .

Use the following URL format:

https://tasking-manager-tm4-production-api.hotosm.org/api/v2/projects/PROJECT_ID/tasks/?as_file=true&format=geojson

For example, for project #12658:

https://tasking-manager-tm4-production-api.hotosm.org/api/v2/projects/12658/tasks/?as_file=true&format=geojson

Make sure to tick the option "Download as new layer."

Don’t worry about memorizing the URL—JOSM will remember it, and you’ll only need to edit the PROJECT_ID part.

I import the taskgrid because, in this project, we are supposed to map buildings outside the task boundary. Since we don’t want to map buildings inside other tasks, the taskgrid helps us determine whether they are inside or not.

Map Paint Styles

Map paint styles are useful tools that can change how the map appears. I created a custom style for my preferences, and you can do the same using MapCSS.

  • Missing Maps / Youth Mappers Validators Checks: Highlighting shared nodes
  • Inline Validation: Highlighting overlapping buildings
  • Apple Paint Style: Can do basically the same as the two styles mentioned above but also much much more. So you can choose paint style on your visual preferences.
  • TM Tasks Grid: Shows task numbers and their states (mapped, validated, not mapped, etc.)
  • New Modified Buildings: Highlights edited and created buildings

Search

Edit › Search {Ctrl+F}. A very useful tool for validation. If you frequently use certain search strings, you can turn them into buttons on your toolbar:

Tick the Add toolbar button option in the Search window. You can customize button icons via Edit › Preferences {F12} › Toolbar or Configure Toolbar.

Some recommended search strings:

  • building:-yes: Select all buildings without building=yes.
  • building=yes nodes:5-20: Select buildings with 5–20 nodes (useful for catching circular buildings).
  • type:way: Select task borders being validated.
  • area[building] > node:connection: Use MapCSS selector in the search panel . Highlights shared nodes for easier selection.

Workflow

In Tasking Manager, I select multiple tasks (filtered by user for better feedback). After locking tasks, I check for mapper comments or questions.

Steps in JOSM:

  1. Import the taskgrid (File › Open Location).
  2. Run validation searches and add flagged results to the Todo list.
  3. Check task boundaries and surrounding areas for unmapped buildings.

After validating all tasks, I leave feedback, pointing out mistakes with examples or screenshots. Comments should be clear and concise, with references to specific tasks if needed.
